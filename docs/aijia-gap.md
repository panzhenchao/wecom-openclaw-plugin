# AIjia 企业微信能力差距

对比对象：OpenClaw 官方企业微信插件（WecomTeam/wecom-openclaw-plugin）
对比日期：2026-06-16

这份文档只描述用户能感知到的功能差异，不展开代码实现。

## 结论

AIjia 目前已经能完成企业微信机器人的基础接入：扫码或手填机器人信息后，私聊和群聊里的文字消息可以进入 AIjia，AI 能回复文字；用户发送图片或文件时，AIjia 会尝试接收并交给 AI 处理。

但和 OpenClaw 相比，AIjia 仍是“基础机器人聊天”阶段。OpenClaw 覆盖了更多真实企业微信使用场景，例如自建应用、回调地址接入、语音/视频/图文混排、引用消息、按钮卡片、多账号、群/私聊权限控制、断线兜底和主动发送文件。

## 8 个维度总览

| 维度 | AIjia 当前状态 | OpenClaw 状态 | 用户能感知到的差异 |
|---|---|---|---|
| 消息收发（文字/媒体） | 部分支持 | 支持更完整 | AIjia 可以收发文字，也能接收图片和文件；但用户发语音、视频、图文混排时，AIjia 不能完整理解或处理。AIjia 也还不能主动把图片、文件发回企业微信。 |
| 会话/线程管理 | 部分支持 | 支持更完整 | AIjia 能区分不同私聊和群聊；但 OpenClaw 可以把不同用户或群隔离成更独立的工作上下文，更适合多人长期使用。 |
| 注册与鉴权 | 部分支持 | 支持更完整 | AIjia 支持扫码或手填机器人信息；OpenClaw 还支持企业微信自建应用、回调密钥、企业凭证等更完整的企业接入方式。 |
| 回调地址和扫码流程 | 部分支持 | 支持更完整 | AIjia 主要依赖机器人长连接和扫码配置；OpenClaw 还支持把企业微信后台配置成“回调地址”来接收消息和事件。 |
| 消息排队、回复和注册体验 | 部分支持 | 支持更完整 | AIjia 可以把企业微信消息送进 AI 对话并回复；但回复通常是生成完再一次性发出。OpenClaw 支持“正在思考”、流式更新和更多回复兜底。 |
| 错误处理与重试 | 部分支持 | 支持更完整 | AIjia 能提示连接异常或需要重新配置；OpenClaw 在机器人不可用时，还能尝试用自建应用继续发送消息。 |
| 多账号支持 | 未支持 | 支持 | AIjia 当前只适合绑定一个企业微信机器人；OpenClaw 可以同时配置多个企业微信账号或机器人。 |
| 群聊 vs 私聊 | 部分支持 | 支持更完整 | AIjia 可以处理群聊和私聊的基础消息；OpenClaw 可以进一步控制哪些群、哪些成员、哪些私聊用户允许使用。 |

## 主要差距清单

| 功能点 | OpenClaw 有 | AIjia 有 | 用户能感知到的问题 | 优先级 |
|---|---|---|---|---|
| 企业微信回调地址接入 | 有 | 无 | 用户如果只想在企业微信后台填一个回调地址接入，AIjia 当前做不到。 | 高 |
| 企业微信自建应用模式 | 有 | 无 | 用户不能把 AIjia 当成企业微信自建应用使用，也不能通过自建应用向成员、部门或群主动发消息。 | 高 |
| 语音消息 | 有 | 无 | 用户在企业微信发语音，OpenClaw 可以使用语音转文字内容；AIjia 当前不能稳定进入 AI 对话。 | 高 |
| 视频消息 | 有 | 无 | 用户发视频时，AIjia 当前不能像 OpenClaw 一样把视频作为可处理附件交给 AI。 | 高 |
| 图文混排消息 | 有 | 无 | 用户一次发送文字加多张图片时，AIjia 可能只能处理其中一部分或直接忽略。 | 高 |
| 引用/回复某条消息 | 有 | 无 | 用户在群里引用某条消息问“帮我看这个”，AIjia 不一定知道“这个”指哪条消息。 | 中 |
| 边生成边显示 | 有 | 无 | OpenClaw 能让用户看到“正在思考”或逐步更新；AIjia 通常等 AI 完整生成后才发一条回复。 | 中 |
| 按钮卡片/投票卡片/通知卡片 | 有 | 无 | AIjia 只能发文字，不能在企业微信里展示可点击按钮、投票或结构化通知。 | 中 |
| 主动发送图片和文件 | 有 | 无 | AI 想把生成的图片、文档或附件发回企业微信时，AIjia 当前做不到或只能退化成文字说明。 | 高 |
| 多企业微信账号/多机器人 | 有 | 无 | 一个用户或团队想同时接多个企业微信机器人时，AIjia 当前只能维护一个。 | 高 |
| 群聊白名单 | 有 | 无 | 管理员不能只允许某些群使用 AIjia；机器人被拉进不该服务的群时缺少产品化控制。 | 中 |
| 群成员白名单 | 有 | 无 | 管理员不能限制“只有群里的某几个人能触发机器人”。 | 中 |
| 私聊白名单/配对审批 | 有 | 无 | 管理员不能控制哪些员工可以私聊使用机器人。 | 中 |
| 机器人断线后的发送兜底 | 有 | 无 | 机器人长连接异常时，OpenClaw 可尝试走自建应用继续发；AIjia 当前更容易表现为发送失败或等待重连。 | 高 |
| 定时广播到部门、标签或群 | 有 | 无 | OpenClaw 可以把定时任务结果发到部门、标签或群；AIjia 当前企业微信渠道不支持这种交付方式。 | 中 |
| 按用户/群自动隔离 AI 助手 | 有 | 无 | OpenClaw 可以让不同用户或群使用独立助手上下文；AIjia 当前更多是普通会话映射。 | 低 |
| 企业微信工具能力 | 有 | 无 | OpenClaw 带企业微信相关工具和技能，例如联系人、日程、待办、文档等；AIjia 当前企业微信渠道主要是消息入口。 | 低 |

## 建议补齐顺序

1. 先补高频基础能力：回调地址接入、自建应用模式、语音/视频/图文混排、主动发送图片和文件、多账号。
2. 再补体验和管理能力：引用消息、边生成边显示、按钮卡片、群/私聊白名单、断线发送兜底。
3. 最后补增强能力：按用户/群隔离助手、企业微信工具和技能。

## 复核重点

- 文档使用产品语言描述差距，避免实现细节。
- 差距项均对应用户可感知的功能缺口。
- 高优先级集中在“接入方式、媒体处理、主动发送、多账号、断线兜底”这些会直接影响真实使用的能力。
